Add a message flow

(Category: Process Modeling)

“Messages” have a definition specific to BPMN - these are not messages sent by email or chat. (To send email, see email connectors).

Message events are used in a process to carry information from one process to another (since transitions / process flow cannot cross process boundaries in BPMN). Messages cannot be sent within a process – only between processes.

Messages are useful to synchronize two processes, and can also be used to correlate two process instances.

Messages are unique; that is, a single message is sent and a target can receive a single message. (To send information to multiple receivers at once, or within a pool, use signals.)

Messages require configuration called data mapping between the message sent and the message received in the different processes involved. Data mapping can also include configuration of message correlation.

There are five types of message events: throw message, catch message, start message, end message, and boundary message.

There are two ways to add messages to a process design in Bonita Studio:

  • using drag and drop from the design palette
  • drag and drop via the context palette on an element
